# Critical Anti-Patterns (NÃO repetir — descobertos por falha real nesta sessão)

| Padrão | Como se manifestou | Severidade | Mitigação estrutural |
|--------|--------------------|------------|----------------------|
| grep no código legado | `grep` aqui é **ugrep** e os arquivos são **ISO-8859** → ele os ignora como binário e dá **falso negativo** (quase concluí busca de segurança "limpa" que não era) | blocking | Sempre `grep -a` + `export LC_ALL=C`; validar com token conhecido (`file` mostra "ISO-8859 text") |
| `pkill -f` matando o próprio shell | `pkill -f 'receituario-backend.jar'` casa com o próprio comando bash → saída 144 | blocking | Usar classe de caractere: `pkill -f 'receituari[o]-backend.jar'` (ou matar por PID file) |
| `.env` DB_PASSWORD | valor começa com `$` → `source`/export expande e quebra o psql | advisory | Ler literal: `grep '^[[:space:]]*DB_PASSWORD=' .env | sed -E 's/^[[:space:]]*DB_PASSWORD=//'` |
| Geração de PDF | sem pandoc/LaTeX/Chromium; puppeteer não extrai (sem `unzip`); pdfmake **0.3.x** quebra server-side | advisory | Usar **pdfmake 0.2.7** + `new (require('pdfmake'))(fonts).createPdfKitDocument(dd).pipe(...)`; fontes Roboto extraídas de `build/vfs_fonts` |
